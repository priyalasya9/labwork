# ‚úÖ **PART 1 ‚Äî Prerequisites**

Before starting, make sure you have:

* Jenkins installed & running
* Git installed on the Jenkins machine
* SMTP email setup (Email Extension Plugin + Gmail App Password, etc.)
* A GitHub repository URL (public or private)

# ‚úÖ **PART 2 ‚Äî Setup Email Extension Plugin (Only once)**

You have already detailed these steps, but I will summarize cleanly:

### **Step 1 ‚Äî Install Email Extension Plugin**

# üü© **PART 4 ‚Äî Create a Jenkins Pipeline From a GitHub Repo**

### **Step 4 ‚Äî Create a New Pipeline Job**

1. Go to **Jenkins Dashboard**
2. Click **New Item**
3. Enter a name:
   **MyGitHubPipeline**
4. Select **Pipeline**
5. Click **OK**

---

### **Step 5 ‚Äî Connect Pipeline to GitHub Repository**

Inside the job:

1. Go to **Build Triggers**

2. Enable:
   ‚úî **GitHub hook trigger for GITScm polling** (if using webhook)
   OR
   ‚úî **Poll SCM** ‚Üí `H/5 * * * *` (poll every 5 minutes)

3. Go to **Pipeline** section

4. In "Definition", select:

   **Pipeline script from SCM**

5. Now fill:

| Field             | Value                                                                                       |
| ----------------- | ------------------------------------------------------------------------------------------- |
| SCM               | Git                                                                                         |
| Repository URL    | your GitHub URL (e.g. [https://github.com/user/repo.git](https://github.com/user/repo.git)) |
| Credentials       | Add ‚Üí GitHub token (if private repo)                                                        |
| Branches to build | main or master                                                                              |

6. Script path:

```
Jenkinsfile
```

7. Click **Save**

---

# üü• **PART 5 ‚Äî Add Jenkinsfile to Your GitHub Repo**

Create a file called **Jenkinsfile** in your repo:

```groovy
pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                echo "Running the build..."
            }
        }
    }

    post {
        success {
            emailext(
                subject: "SUCCESS: ${env.JOB_NAME} #${env.BUILD_NUMBER}",
                body: "Build completed successfully for job: ${env.JOB_NAME}",
                to: "lasyapriyasoma@gmail.com"
            )
        }
        failure {
            emailext(
                subject: "FAILURE: ${env.JOB_NAME} #${env.BUILD_NUMBER}",
                body: "Build failed! Please check Jenkins.",
                to: "lasyapriyasoma@gmail.com"
            )
        }
    }
}
```

Commit + push this file.

3. Jenkins fetches your GitHub repo & runs the Jenkinsfile
